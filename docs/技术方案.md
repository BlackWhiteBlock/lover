# Lover MVP 技术方案

## 1. 目标与范围

Lover 是面向情侣的双人私密空间。MVP 建立以下核心闭环：

1. 手机验证码登录与安全会话；
2. 创建邀请码并绑定两名用户；
3. 展示相爱天数和近期影像；
4. 上传、浏览照片与视频；
5. 创建年度纪念日和里程碑；
6. 发送即时信与时间胶囊；
7. 退出登录以及双方确认解绑。

空间装扮、共同愿望单、推送通知、年度报告和桌面小组件属于后续阶段。

## 2. 总体架构

```text
Android 原生客户端
  Kotlin + Jetpack Compose
           │ HTTPS / JSON
           ▼
Fastify 模块化单体 API
           │
    ┌──────┼───────────┐
    ▼      ▼           ▼
PostgreSQL 七牛私有存储 阿里云短信
```

当前采用模块化单体，不在 MVP 阶段拆分微服务。该方案复用了 Family 项目已验证的
Fastify、PostgreSQL、短信和对象存储经验，但重新设计了对称情侣模型，未引入
`dad/mom/child_id` 等亲子领域约束。

## 3. Android 客户端

### 3.1 技术栈

- Kotlin、JDK 21
- Jetpack Compose、Material 3
- Hilt
- Retrofit、OkHttp、Kotlin Serialization
- Coroutines、Flow
- DataStore
- Coil
- Media3 ExoPlayer
- Android Photo Picker

最低版本 API 26，编译与目标版本 API 36。

### 3.2 分层

```text
Compose Screen
    ↓
ViewModel
    ↓
AppRepository
    ├── ApiService
    ├── TokenStore
    ├── AssetUploader
    └── Media Resolver
```

UI 使用单向数据流。`ApiService` 集中声明服务端契约；`TokenStore` 使用 DataStore
保存访问令牌、刷新令牌、用户信息和最近一次服务端缓存。

### 3.3 会话管理

- Access Token 自动附加到 Lover API 同源请求；
- 401 时由 OkHttp Authenticator 单飞刷新令牌并重试；
- Refresh Token 每次使用后由服务端轮换；
- 第三方对象存储请求使用独立客户端，禁止携带 Lover Token；
- 冷启动先加载本地会话，再请求 `/api/me` 和业务数据；
- 网络暂时不可用时展示缓存，认证失效时返回登录页。

### 3.4 媒体

Photo Picker 支持图片和视频。源文件通过 `RequestBody` 流式读取，不整体加载到内存：

- 图片上限 30 MB；
- 视频上限 200 MB；
- 视频通过 `MediaMetadataRetriever` 提取 JPEG 封面；
- 视频和封面分别上传并完成校验；
- 服务端记录同时引用视频资产和封面资产；
- 图片由 Coil 展示，视频由 ExoPlayer 播放。

## 4. 服务端

### 4.1 技术栈

- Node.js 20+
- TypeScript
- Fastify 5
- Zod
- PostgreSQL 14+
- JWT、bcrypt
- 阿里云短信
- 七牛 Kodo 私有空间
- OpenAPI / Swagger

业务按 `auth`、`couples`、`assets`、`media`、`anniversaries`、`letters` 和
`bootstrap` 模块拆分。

### 4.2 核心数据表

```text
users
sms_codes
auth_sessions

couple_spaces
couple_members
couple_invites
unbinding_requests

media_assets
media_items
anniversaries
letters
```

关键约束：

- 一个用户最多属于一个 active 情侣空间；
- 一个空间最多有两名 active 成员；
- 双方权限对称，不使用性别角色；
- 所有业务记录均通过服务端解析的 `space_id` 隔离；
- 客户端不能通过提交其他 `space_id` 访问跨空间数据。

### 4.3 登录与绑定

验证码经 bcrypt 哈希后保存，并具有过期、消费和发送冷却状态。登录成功后签发短期
Access Token 与可轮换 Refresh Token，数据库只保存 Refresh Token 的 SHA-256 摘要。

邀请码只保存哈希值，有效期为七天且一次性使用。接受邀请在数据库事务中完成，并通过
行锁、唯一索引和成员数量约束处理并发绑定。

### 4.4 对象存储

开发环境可以使用受签名接口保护的本地存储；生产环境强制使用七牛私有 Bucket：

1. 客户端申请上传凭证；
2. 服务端生成空间隔离的 object key；
3. 客户端直传 Local 或七牛；
4. 客户端只提交 asset ID 请求完成；
5. 服务端从数据库取得真实 object key，通过存储 Provider 校验大小和 MIME；
6. 业务记录只能引用 ready 状态的资产；
7. 阅读时签发短期私有 URL。

七牛上传策略锁定精确 `bucket:key`，启用 `insertOnly` 和 `forceSaveKey`。object key
解析拒绝绝对路径、反斜线、查询参数以及 `.`、`..` 等路径穿越形式。

### 4.5 时间胶囊

- 客户端将所选日期按本地午夜转换为带时区偏移的 ISO 时间；
- 服务端以 `unlock_at` 和数据库时间作为唯一解锁依据；
- 解锁前列表和详情响应均不包含 `content` 与 `summary`；
- 客户端只使用服务端 `isUnlocked`，不能通过修改手机时间提前查看；
- 到期读取不依赖定时任务，避免任务延迟阻止正常解锁。

### 4.6 解绑

一名成员发起解绑申请，另一名成员确认后才执行。确认时双方 membership 转为 inactive，
空间转为 dissolved，未使用邀请同时取消。历史内容保留但双方立即失去访问权限。

## 5. 安全策略

- 生产环境禁止开发短信 Provider和默认密钥；
- 生产环境禁止 Local Storage；
- Android Release 禁止明文 HTTP；
- API 输入使用严格 Zod 校验；
- 验证码、登录、邀请和上传凭证接口限流；
- 媒体签发前再次检查当前 active membership；
- 5xx 响应不暴露 SQL 和内部堆栈；
- 所有存储和业务查询均进行空间归属检查；
- Swagger 和日志在生产环境应由反向代理控制访问范围。

单进程限流器适合当前 PM2 单实例部署。扩展为多副本时，应将限流状态迁移到共享 Redis。

## 6. API 设计

主要接口：

```text
POST /api/auth/sms/send
POST /api/auth/sms/login
POST /api/auth/refresh
POST /api/auth/logout
GET  /api/me

GET|PATCH /api/couple-space
POST /api/couple-invites
POST /api/couple-invites/accept
POST /api/couple-space/unbinding

GET /api/bootstrap

POST /api/media-assets/token
POST /api/media-assets/complete
POST /api/media-assets/:assetId/sign
GET|POST|PATCH|DELETE /api/media

GET|POST|PATCH|DELETE /api/anniversaries
GET|POST|PUT|DELETE /api/letters
```

完整契约由 `backend/src/openapi.ts` 和 Android `ApiService.kt` 共同维护。

## 7. 数据库迁移

迁移采用三位数字前缀 SQL 文件，并记录在 `schema_migrations`：

- `001_initial.sql`：用户、会话、情侣空间和业务表；
- `002_qiniu_storage.sql`：七牛 Provider、Bucket 与对象校验字段。

迁移在事务和 PostgreSQL advisory lock 内执行，避免多实例同时部署导致重复迁移。

- 七牛上传策略和完成校验；
- API 限流、健康与就绪检查。

Android 覆盖：

- Local/七牛上传请求；
- 第三方请求 Token 隔离；
- 图片和视频上传分支；
- ViewModel 输入校验。

GitHub Actions 对每次 PR 执行：

```text
Backend: npm ci → typecheck → test → build
Android: testDebugUnitTest → lintDebug → assembleDebug
```

## 9. 部署

后端生产环境需要：

- PostgreSQL；
- HTTPS API 域名和可信反向代理；
- 独立强随机 JWT/Refresh/Storage 密钥；
- 阿里云短信签名和模板；
- 七牛私有 Bucket、上传域名和下载域名。

Android Release 使用 `LOVER_API_BASE_URL` 注入 HTTPS 地址。签名密钥、短信和存储凭据
不得提交到 Git。
