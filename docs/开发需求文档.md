## 1. 产品概述

### 1.1 产品定位

**Lover** 是一款面向亲密情侣的双人私密空间应用，口号为：

> **lover.** — *Two Hearts One World*

产品核心不是社交广场，而是「两个人的专属世界」：记录相处天数、珍藏影像、纪念重要日子、互写信件（含未来时间胶囊）。

### 1.2 目标用户

| 角色 | 描述 |
|------|------|
| 情侣用户 A / B | 已确立恋爱关系的两名用户，绑定为同一「情侣空间」 |
| 共同操作者 | 双方均可上传媒体、创建纪念日、写信、管理空间 |

### 1.3 产品目标（MVP）

1. 完成双人账号绑定与情侣空间创建  
2. 展示并维护「相爱天数」主里程碑  
3. 支持照片/视频时光相册  
4. 支持纪念日创建与倒计时  
5. 支持即时情书与时间胶囊信封  
6. 提供基础个人/情侣设置入口（装扮、愿望单、隐私、关于）

### 1.4 非目标（本期不做）

- 公开社交 Feed、发现页、陌生人互动  
- 多人家庭/朋友圈扩展  
- 电商、礼物商城  
- AI 自动生成情书（可作为后续增值能力）

## 2. 信息架构与导航

### 2.1 底部主导航（Floating Island）

固定底部悬浮导航栏，共 **5** 个 Tab：

| Tab ID | 中文 | 英文暗示 | 功能 |
|--------|------|----------|------|
| `home` | 空间 | Heart | 首页概览与快捷入口 |
| `gallery` | 时光 | Visual Memories | 相爱时光相册 |
| `anniversaries` | 纪念 | Eternal Dates | 纪念日墙 |
| `letters` | 信封 | Secret Letters | 情书与时间胶囊 |
| `profile` | 我们 | Profile | 情侣资料与设置 |

**交互要求：**

- 选中态：图标放大、品牌玫瑰色高亮，底部有指示圆点动画  
- 未选中态：浅石色（stone）  
- 页面切换带淡入缩放过渡动画  
- 内容区最大宽度约 `md`（约 448px），移动端优先、居中单栏布局  

### 2.2 悬浮操作按钮（FAB）

在 **时光 / 纪念 / 信封** 三个 Tab 显示右下角「+」按钮，用于对应模块的新增操作。

| 当前 Tab | FAB 行为 |
|----------|----------|
| 时光 | 打开上传瞬间（照片/视频） |
| 纪念 | 打开新建纪念日表单 |
| 信封 | 打开写信（即时 / 时间胶囊） |

首页已有「写情书 / 存瞬间」快捷入口，可不重复展示 FAB。

## 3. 功能需求详述

### 3.1 首页 —「空间」

#### 3.1.1 品牌头

- 主标题：`lover.`（衬线体、大字号、弱化玫瑰色）  
- 副文案：`Two Hearts One World`（小号全大写、宽字距）

#### 3.1.2 相爱里程碑（Milestone）

| 字段 | 说明 |
|------|------|
| 相爱天数 | 根据情侣「在一起日期」实时计算（UI 示意如 1314 天） |
| 展示文案 | `Loving Journey` + `{N} 天` |
| 视觉 | 大心形动画 + 毛玻璃卡片 |

**业务规则：**

- 天数 = 今天日期 − 情侣建立日期（含当天，或按产品约定统一口径）  
- 绑定后首页必须展示；未设置在一起日期时引导补全  

#### 3.1.3 快捷操作

两列入口：

| 按钮 | 行为 |
|------|------|
| 写情书 / Write Love | 跳转信封模块并进入写信 |
| 存瞬间 / Capture Now | 跳转时光模块并进入上传 |

#### 3.1.4 近期掠影

- 横向滚动媒体缩略图（约 3+ 条）  
- 数据来源：时光相册按时间倒序的最新条目  
- 点击进入对应媒体详情  

---

### 3.2 时光 —「相爱时光」

#### 3.2.1 列表展示

双列瀑布/网格（设计为 `aspect 4:5` 圆角卡片），每项包含：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| type | `image` \| `video` | 是 | 媒体类型 |
| url | string | 是 | 资源地址 |
| thumbnail | string | 视频建议必填 | 视频封面 |
| caption | string | 否 | 描述文案，叠在卡片底部 |
| date | date | 是 | 拍摄/记录日期 |

**交互：**

- 视频项显示半透明播放按钮覆盖层  
- 点击预览全屏：图片可缩放；视频可播放/暂停  
- 标题区：`相爱时光` / `Visual Memories`，右上角「+」上传  

#### 3.2.2 上传瞬间（新增）

**入口：** 页头「+」、首页「存瞬间」、对应 FAB  

**表单字段：**

| 字段 | 校验 |
|------|------|
| 媒体文件 | 支持图片（jpg/png/webp/heic）、视频（mp4/mov）；需限制单文件大小与时长 |
| 说明 caption | 可选，建议 ≤ 200 字 |
| 日期 | 默认今天，可改 |

**业务规则：**

- 仅情侣双方可见  
- 上传成功后刷新列表与首页「近期掠影」  
- 失败需明确错误提示（网络、体积、权限）  

---

### 3.3 纪念 —「爱的纪念日」

#### 3.3.1 列表展示

大图横向卡片，包含：

| 字段 | 说明 |
|------|------|
| title | 纪念日名称，如「相遇一千天」「正式在一起」 |
| date | 纪念日期 |
| image | 封面图 |
| type | `milestone`（里程碑） / `yearly`（年度纪念） |
| countdown | 距下次发生还剩天数 |

**标签文案：**

- `milestone` →「里程碑」  
- `yearly` →「年度纪念」  

**倒计时规则（建议）：**

- `yearly`：按每年循环，取下一个周年日  
- `milestone`：按设定的目标日计算；若已过期则显示「已达成」或转为回顾态（需产品确认）  

列表底部提供虚线按钮：「添加新的纪念日」。

#### 3.3.2 新建纪念日

| 字段 | 必填 | 说明 |
|------|------|------|
| 标题 | 是 | ≤ 30 字 |
| 日期 | 是 | 年-月-日 |
| 类型 | 是 | 里程碑 / 年度纪念 |
| 封面 | 否 | 可从相册选或拍摄 |

---

### 3.4 信封 —「爱的信封」

#### 3.4.1 信件模型

| 字段 | 说明 |
|------|------|
| sender | 发送方（A / B 或昵称） |
| title | 标题 |
| content | 正文 |
| type | `instant`（即时） / `capsule`（时间胶囊） |
| unlockDate | 时间胶囊解锁日期（仅 capsule） |
| isUnlocked | 是否已解锁 |
| createDate | 创建日期 |

#### 3.4.2 列表交互

- **Instant**：可直接阅读正文摘要（行数截断），点击进详情  
- **Time Capsule 未解锁**：显示锁图标 +「预计解锁时间」，正文不可见  
- **Time Capsule 已解锁**：行为同即时信  

页头右侧：

- 历史（History）：筛选/查看已读、往年信件（可做筛选面板）  
- 写信（Mail）：创建新信件  

#### 3.4.3 写信

| 字段 | 必填 | 说明 |
|------|------|------|
| 标题 | 是 | |
| 正文 | 是 | 建议支持多段文本 |
| 类型 | 是 | Instant / Time Capsule |
| 解锁日期 | 胶囊必填 | 必须晚于今天 |

**业务规则：**

- 胶囊在解锁日前，接收方与发送方均不可查看正文（或发送方可见摘要——需产品二选一，**默认双方都不可见以增强仪式感**）  
- 到解锁日 00:00（按用户时区或统一 UTC+8）自动解锁，并可推送通知  
- 信件仅情侣双方可见  

---

### 3.5 我们 — Profile

#### 3.5.1 情侣身份区

- 双方头像叠放（示意字母 A / B）  
- 标题：「我们的小宇宙」  
- 副文案：`Established {YYYY.MM.DD}`（情侣空间建立/在一起日期）  

#### 3.5.2 设置入口（设计已露出，需落地）

| 菜单 | MVP 范围 | 说明 |
|------|----------|------|
| 空间装扮 | P1 | 背景色/主题皮肤/昵称展示方式 |
| 共同愿望单 | P1 | 共享待办/愿望列表（增删改状态） |
| 隐私控制 | P0 | 退出登录、解绑、生物识别锁、数据导出/注销（最小集先做解绑与退出） |
| 关于 Lover | P0 | 版本号、协议、反馈入口 |

## 4. 账号与情侣绑定（设计隐含，必须补齐）

UI 稿未画出登录页，但产品运行必需，建议作为 **P0** 基础设施：

### 4.1 账号

- 手机号 / 邮箱 + 验证码登录（实现时二选一或都支持）  
- 用户资料：昵称、头像  

### 4.2 情侣空间绑定

| 步骤 | 说明 |
|------|------|
| 创建邀请 | A 生成邀请码 / 深链 / 二维码 |
| 接受绑定 | B 输入邀请码完成配对 |
| 空间属性 | `coupleId`、双方 `userId`、`togetherDate`、空间昵称 |

**规则：**

- 一对用户仅属于一个情侣空间（MVP）  
- 解绑需双方确认或冷静期（产品确认）  
- 解绑后媒体与信件可见策略需明确（保留给原主人 / 清空）

## 6. 界面与体验规范（对齐设计稿）

### 6.1 视觉关键词

- 温暖轻柔：底色近似 `#fffcfb`  
- 主色：玫瑰系（rose-300 / rose-400）  
- 辅色：石色（stone）、淡橙氛围光斑  
- 大圆角卡片（约 2rem–3.5rem）  
- 毛玻璃与轻阴影，避免厚重拟物  
- 标题使用衬线体；正文轻量无衬线  

### 6.2 动效

最少保留以下动效（设计稿已体现）：

1. Tab 切换淡入与轻微缩放  
2. 导航选中圆点 `layout` 位移动画  
3. 背景光斑缓慢漂浮  
4. 里程碑心形呼吸缩放  
5. FAB 出现时的旋转缩放  

### 6.3 适配

- 移动端优先（375–430 宽度）  
- 桌面访问时居中窄栏展示  
- 底部导航与安全区（Home Indicator）留白  
- 列表滚动区域避开底部导航遮挡（底部 padding 预留）

## 7. 非功能需求

| 类别 | 要求 |
|------|------|
| 隐私 | 默认双人私密空间，无公开检索；媒体与信件加密传输（HTTPS） |
| 性能 | 首页首屏骨架/占位；相册懒加载与缩略图 |
| 存储 | 图片转码压缩；视频转码与封面抽取 |
| 可靠性 | 上传失败可重试；关键日期变更服务端统一计算 |
| 通知 | 纪念日提醒、胶囊解锁提醒（推送可选，SMS/邮件次之） |
| 安全 | 鉴权 Token；情侣空间行级权限校验（禁止越权读他人空间） |
| 国际化 | MVP 以简体中文为主，英文副文案可静态保留 |

## 8. 模块优先级与里程碑建议

### Phase 0 — 骨架与设计还原

- [ ] 五 Tab 导航与页面框架  
- [ ] 设计 Token（色、圆角、字体、动效）落地  
- [ ] 静态数据跑通 UI 交互  

### Phase 1 — MVP 核心闭环

- [ ] 登录注册  
- [ ] 情侣绑定与 `togetherDate`  
- [ ] 首页天数 + 近期掠影  
- [ ] 时光上传/列表/预览  
- [ ] 纪念日 CRUD + 倒计时

- [ ] 空间装扮

### Phase 3 — 增长与留存（可选）

- [ ] 年度回忆报告  
- [ ] 纪念日卡片分享图（分享时脱敏）

## 9. 验收标准（关键路径）

1. **绑定**：A 邀请 B，双方进入同一空间，首页显示正确相爱天数  
2. **存瞬间**：上传一张照片、一条视频，时光列表与首页掠影同步出现  
3. **纪念日**：新建年度纪念日，倒计时天数与日历一致  
4. **写信**：发送即时信，对方可读；发送未来胶囊，未到期不可读，到期后可读  
5. **导航**：五 Tab 切换流畅，FAB 仅在时光/纪念/信封出现且行为正确  
6. **视觉**：整体风格与 `UI-Figma` 一致（色调、圆角、岛屿导航、衬线标题）

## 10. 待产品确认项（Open Questions）

设计稿为高保真交互示意，以下项需在开发前拍板：

1. 端形态：纯 H5 / PWA / iOS+Android 原生（Flutter/React Native）？  
2. 时间胶囊：发送方在解锁前是否可看自己写的正文？  
3. 解绑后历史内容归属策略？  
4. 媒体审核与敏感内容策略（双人私密是否仍需基础风控）？  
5. 「共同愿望单」「空间装扮」是否纳入 MVP，还是 Phase 2？  
6. 登录方式与实名强度（仅邮箱 vs 手机号）？
